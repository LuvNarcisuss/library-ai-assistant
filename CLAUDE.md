# Library AI Assistant - 项目配置

## 项目目标

构建一个基于 RAG（检索增强生成）技术的**对话机器人及智能体开发平台**，解决企业知识资产分散、信息检索效率低下的问题，提供从后端数据处理（以RAG为核心进行知识库维护）、混合搜索、智能路由到模型微调的全链路、生产级解决方案。

## 产品愿景

成为业界领先的**对话机器人及智能体开发平台**，不仅仅是一个应用，更是一个**高内聚、低耦合的开发脚手架和最佳实践集合**。

## 选题要求

完成机器人智能对话，例如：塞尔达对话机器人、XX大学AI对话机器人（集成学生手册、学生新闻等并融合请假等智能体开发）、其他有意思的AI机器人。

## 技术栈

### 后端
- **Web 框架**: FastAPI (异步、高性能)
- **服务器**: Uvicorn (asyncio 驱动)
- **数据库**: SQLite + SQLAlchemy ORM
- **向量数据库**: ChromaDB (嵌入式，无需外部服务)
- **Embedding**: BAAI/bge-small-zh-v1.5 (384 维中文 Embedding)
- **文档解析**: python-docx / pypdf
- **文本切分**: LangChain Text Splitters (RecursiveCharacterTextSplitter)
- **认证**: python-jose (JWT Token)

### 前端
- **框架**: Vue 3 + Vite
- **UI 库**: Element Plus
- **HTTP**: Axios
- **路由**: Vue Router
- **Markdown**: markdown-it

## 核心功能模块

### 一、后台知识库管理平台

#### 模块一：知识库管理
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 知识库创建 | 支持自定义知识库名称、描述、所属部门、负责人，指定默认的 Embedding 模型（如 bge-small-zh-v1.5） | 高 | 关联 Chroma 向量库目录（按知识库名隔离） |
| 知识库列表 | 展示所有知识库的基本信息（名称、文档数量、创建时间、负责人、状态），支持按名称 / 部门 / 创建时间筛选 | 高 | 读取 Chroma 元数据 + 本地配置文件 |
| 知识库编辑 | 支持修改知识库名称、描述、负责人，切换 Embedding 模型（如从 small 切到 base） | 中 | 更新配置文件，不影响已有向量 |
| 知识库删除 | 支持逻辑删除 / 物理删除，删除前校验是否有引用，物理删除需清空对应 Chroma 向量库 | 中 | 删除 Chroma 目录 + 数据库记录 |
| 知识库克隆 | 复制已有知识库的文档和配置，生成新知识库 | 低 | 复制 Chroma 向量库 + 元数据 |

#### 模块二：文档管理
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 文档批量上传 | 支持上传 txt、docx、pdf、md 等格式，支持拖拽 / 文件夹上传，显示上传进度 | 高 | python-multipart 接收文件，分块读取大文件 |
| 文档格式解析 | 自动解析上传的文档，提取文本内容（PDF 提取文字、docx 解析段落），保留基本格式（标题、列表） | 高 | 用 python-docx/PyPDF2 解析，清洗乱码 / 特殊字符 |
| 文档拆分配置 | 支持自定义文本拆分参数（chunk_size、chunk_overlap），预览拆分后的文本块 | 高 | 对接 langchain_text_splitters，可视化拆分结果 |
| 文档列表 | 展示知识库下所有文档（名称、大小、上传时间、拆分块数、状态），支持按名称 / 上传时间筛选 | 高 | 读取本地文件元数据 + Chroma 文本块计数 |
| 文档预览 | 在线预览原始文档内容和拆分后的文本块，支持搜索文本块内容 | 中 | 前端渲染纯文本，高亮匹配关键词 |
| 文档编辑 | 支持手动修改文档文本内容，修改后自动重新生成向量 | 中 | 重新拆分 + 调用 Embedding 模型更新向量 |
| 文档删除 | 删除文档同时删除对应的向量数据，支持批量删除 | 高 | 删除 Chroma 中对应文档的所有向量 |
| 文档版本管理 | 记录文档的修改历史，支持回滚到历史版本 | 低 | 本地保存文档历史版本，版本切换时重新生成向量 |

#### 模块三：向量管理
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 向量生成 | 文档上传 / 修改后自动生成向量，支持手动触发重新生成，显示生成进度 | 高 | 调用 bge-small-zh-v1.5或其他模型 批量生成向量，进度条实时反馈 |
| 向量预览 | 查看文本块对应的向量维度、相似度得分（仅调试用） | 低 | 读取 Chroma 向量数据，格式化展示 |
| 向量优化 | 支持向量归一化、去重（删除重复文本块的向量） | 中 | 基于文本相似度去重，保留高分向量 |

#### 模块四：系统管理模块
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 模型配置 | 管理 Embedding 模型列表（名称、路径、维度、描述），设置默认模型 | 高 | 读取本地模型目录，关联 Embedding 类 |
| 系统参数 | 配置默认文本拆分参数、文件上传大小限制、向量库存储路径、接口请求超时时间 | 高 | 写入全局配置文件（yaml/json） |
| 缓存配置 | 配置向量缓存策略、接口响应缓存时间 | 低 | 对接 Redis 缓存（可选） |

#### 模块五：权限管理
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 用户管理 | 支持创建 / 编辑 / 删除用户，设置用户名、密码、邮箱、所属部门、状态（启用 / 禁用） | 高 | 本地数据库存储用户信息，密码加密 |
| 角色管理 | 预设角色（超级管理员、知识库管理员、普通用户），支持自定义角色 | 高 | 定义角色权限矩阵（如超级管理员可操作所有知识库） |
| 权限分配 | 给用户分配角色，支持精细化权限（如仅允许编辑指定知识库） | 中 | 基于角色的访问控制（RBAC），接口层校验权限 |
| 登录认证 | 账号密码登录，支持记住登录状态，登录失败次数限制（防止暴力破解） | 高 | 接口请求携带 token 校验 |

#### 模块六：配置管理
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 回复速度配置 | 设置检索超时时间、答案生成超时时间，超时返回友好提示 | 中 | FastAPI 接口超时配置，后端设置超时中断 |
| 回答长度限制 | 限制单轮答案的最大长度（如 500 字），避免返回过长内容 | 中 | 答案生成后截断，保留核心信息 |
| 检索参数配置 | 全局 / 会话级配置检索参数（top_k、相似度阈值），支持实时调整 | 中 | 对接检索接口参数，会话级参数覆盖全局参数 |
| 多轮上下文长度 | 设置上下文最大 token 数 / 轮数，超出自动清理最早的对话 | 高 | 计算对话文本 token 数，超出则截断上下文 |

#### 模块七：对话日志与分析
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 对话日志记录 | 记录所有对话（用户提问、机器人回答、检索参数、来源知识库、时间、用户 ID） | 高 | 写入数据库（SQLite/MySQL），按会话 ID 归档 |
| 日志查询与筛选 | 支持按用户 ID、时间、关键词、知识库筛选对话日志，查看单条日志详情 | 中 | 数据库多条件查询，展示完整对话链路 |
| 对话数据分析 | 统计热门问题、检索成功率、平均响应时间、用户活跃度，生成可视化报表 | 低 | 聚合对话日志，生成柱状图 / 折线图 / 词云 |
| 日志导出与清理 | 支持按时间范围导出对话日志（Excel/CSV），自动 / 手动清理过期日志 | 低 | 读取日志数据格式化导出，定时删除旧日志 |

### 二、对话机器人功能

#### 模块一：基础对话交互
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 单轮问答 | 用户输入自然语言问题，机器人返回基于知识库的精准答案，包含答案内容、来源文档、相似度得分 | 高 | 对接 Chroma 语义检索 + 答案格式化拼接 |
| 多轮对话 | 支持上下文关联，机器人能理解用户的追问（如 "这个文档的发布时间是多少？" 中的 "这个文档"） | 高 | 维护对话上下文列表，检索时携带上下文信息 |
| 对话上下文管理 | 前端展示历史对话记录，支持清空上下文、回滚到指定对话轮次，设置上下文最大保留轮数（如 10 轮） | 高 | 后端用会话 ID 存储上下文，前端缓存历史记录 |
| 多模态输入（可选） | 支持用户上传图片 / 语音提问（如 "图片中的内容是什么？"），自动识别文本后再检索 | 低 | 对接 OCR 识别图片文本、ASR 转语音为文字 |
| 快捷提问 | 预设常见问题模板（如 "知识库中有多少份文档？""葡萄的主要品种有哪些？"），用户一键发送 | 中 | 前端配置快捷问题列表，后端直接匹配检索 |

#### 模块二：语义理解与检索优化
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 意图识别 | 自动识别用户提问意图（如 "查询文档""修改知识库""系统帮助"），非查询意图引导到对应功能 | 中 | 基于关键词 / 轻量分类模型（如 sklearn）识别意图 |
| 问题重写 | 自动优化用户的模糊问题（如 "葡萄怎么样"→"葡萄的口感和品种怎么样"），提升检索准确率 | 中 | 调用轻量 LLM（如 ChatGLM-4）或规则重写问题 |
| 多知识库检索 | 支持选择单个 / 多个知识库检索，默认检索所有有权限的知识库，返回结果标注来源知识库 | 高 | 遍历指定 Chroma 向量库，聚合检索结果 |
| 检索过滤 | 支持按文档类型、上传时间、相似度得分过滤检索结果，仅返回符合条件的答案 | 中 | 检索后过滤结果，设置得分阈值（如≥0.7） |
| 语义纠错 | 自动纠正用户输入的错别字（如 "葡淘"→"葡萄"），不影响检索结果 | 中 | 对接拼音纠错 / 形近字纠错库（如 pycorrector） |

#### 模块三：答案生成与增强
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 答案拼接 | 基于检索到的文本块，自动拼接成通顺、完整的答案，避免碎片化 | 高 | 规则拼接（按相似度排序）+ 简单文本润色 |
| 引用标注 | 答案中标注来源文档（如 "【来源：葡萄品种手册.txt】"），支持点击查看原文 | 高 | 检索结果关联文档 ID，前端跳转文档预览 |
| 答案格式化 | 支持按 markdown 格式返回答案（标题、列表、加粗），前端自动渲染 | 中 | 解析文本块中的格式标记，转换为 markdown |
| LLM 增强答案（可选） | 对接私有化 LLM（如 ChatGLM、Qwen），基于检索结果生成更自然的答案，支持关闭 / 开启 | 中 | RAG 流程：检索→拼接 prompt→调用 LLM→返回答案 |
| 答案反问 / 追问引导 | 答案末尾自动推荐相关问题（如 "您可能还想知道：葡萄的种植条件是什么？"） | 低 | 基于检索结果的关键词生成相关问题 |

#### 模块四：对话规则与兜底
| 功能点 | 详细需求 | 优先级 | 技术适配 |
|--------|----------|--------|----------|
| 知识库兜底 | 检索不到相关结果时，返回预设兜底话术（如 "未在知识库中找到相关答案，请换个问题试试"） | 高 | 检测检索结果数量 / 得分，触发兜底逻辑 |
| 敏感词过滤 | 过滤用户提问和机器人回答中的敏感词，支持自定义敏感词库 | 中 | 敏感词匹配替换（如替换为 *），正则匹配 |
| 对话规则配置 | 配置问答规则（如包含 "帮助" 则返回使用指南，包含 "版本" 则返回系统版本） | 中 | 规则引擎匹配关键词，优先返回规则答案 |
| 拒绝回答配置 | 配置拒绝回答的问题类型（如无关问题、恶意提问），返回拒绝话术 | 中 | 关键词 / 意图识别匹配，触发拒绝逻辑 |

## 开发规范

### 代码风格
- 后端遵循 PEP 8 规范
- 前端遵循 Vue 3 Composition API 最佳实践
- 使用有意义的变量和函数命名
- 重要逻辑添加注释

### Git 提交
- 使用中文提交信息
- 提交格式：`<类型>: <描述>`
- 类型：feat / fix / docs / style / refactor / test / chore

### API 设计
- RESTful 风格
- 统一响应格式：`{ code, message, data }`
- 错误处理：返回明确的错误信息

### 数据库
- 使用 SQLAlchemy ORM 操作 SQLite
- 表名使用复数形式
- 字段使用 snake_case 命名

### Playwright-CLI 使用规范
- **必须在项目根目录使用**：playwright-cli 只能在项目根目录 (`library-ai-assistant/`) 下执行
- **统一存储目录**：所有 playwright-cli 生成的文件（快照、日志、截图）统一存放在 `.playwright-cli/` 目录
- **禁止在子目录使用**：不要在 `backend/`、`frontend/` 或其他子目录中使用 playwright-cli

#### 使用示例
```bash
# ✅ 正确：在项目根目录使用
cd D:\aelm\deeplearning\library-ai-assistant
playwright-cli open "https://example.com"

# ❌ 错误：在子目录使用
cd backend
playwright-cli open "https://example.com"  # 会报错
```

#### 文件存储结构
```
library-ai-assistant/
├─ .playwright-cli/              # playwright-cli 统一存储目录
│  ├─ page-*.yml                 # 页面快照文件
│  ├─ console-*.log              # 控制台日志文件
│  ├─ screenshots/               # 截图文件
│  └─ *.json                     # 状态文件（如登录状态）
├─ playwright-cli.bat            # Windows 封装脚本
└─ playwright-cli-wrapper.sh     # Linux/Mac 封装脚本
```

## 验收标准

### 最小可交付验收标准
- 能上传图书馆文档（txt / pdf / docx）
- 能自动解析文档并提取文本
- 能按参数切分文本块
- 能生成向量并存入 Chroma
- 用户提问后能检索知识库
- 能返回带来源引用的答案
- 有可交互的聊天页面
- 有知识库后台管理页面
- 有用户登录功能
- 有对话日志记录

### 完整功能验收标准
- 知识库管理（创建/编辑/删除/克隆）
- 文档管理（批量上传/格式解析/拆分配置/预览/版本管理）
- 向量管理（生成/预览/优化）
- 系统管理（模型配置/系统参数/缓存配置）
- 权限管理（用户管理/角色管理/权限分配/登录认证）
- 配置管理（回复速度/回答长度/检索参数/多轮上下文）
- 对话日志与分析（日志记录/查询筛选/数据分析/导出清理）
- 基础对话交互（单轮问答/多轮对话/上下文管理/快捷提问）
- 语义理解与检索优化（意图识别/问题重写/多知识库检索/检索过滤/语义纠错）
- 答案生成与增强（答案拼接/引用标注/答案格式化/LLM增强/答案反问）
- 对话规则与兜底（知识库兜底/敏感词过滤/对话规则配置/拒绝回答配置）
